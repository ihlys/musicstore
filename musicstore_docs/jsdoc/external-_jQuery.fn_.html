(static) gridCollection()

Makes container element with items on which it is applied look like a grid. It adds it's own css classes with styles to element and it's items: 


.grid-collection {
  display: flex;
  flex-wrap: wrap;
  justify-content: space-around;
  align-content: flex-start;
}

.grid-collection__item {
  margin: 10px;
  overflow: hidden;
  border-radius: 8px;
  box-shadow: 0 0 8px rgb(178, 178, 178);
}

It also uses module:flex-space-around-helper~FlexSpaceAroundHelper internally to fix align of items in last row.

(static) itemsWithContent(options)

Listens to click events on items of container element on which it is applied and loads content of item to which event belongs. Request that gets items content has URL address specified by data attribute with 'item-page-url' name.

Parameters:
Name Type Description
options object the options object holding:
  1. itemSelector - the CSS selector for items;
  2. prototypingMode - determines if prototyping mode is enabled. If enabled then state of history will not be pushed when sending request for item content.

(static) pageScroller(options)

Loads new page of data asynchronously and appends it to end when container element on which it is used is scrolled to bottom. When next page of data is first requested it uses data attribute with name initial-next-page-url from which it gets URL address for next page. After first request, data attribute is removed by this widget and each subsequent request will be given URL address taken by widget from Link header of previous response.

Parameters:
Name Type Description
options object the options object holding:
  1. scrollPadding - option to determine height from bottom of the container from where widget will start loading next page of data;
  2. prototypingMode - determines if prototyping mode is enabled. If enabled then after first request widget will not read Link header value for next page url and will set it to null internally.

(static) textfit(options)

Changes css styles of text element to fit it's container width. It uses jquery-textfill plugin under the hood, but besides font size it is also manipulates "white-space" css style to configure line wrapping. It works like follows:

  1. It takes same options object that jquery-textfill supports;
  2. It initially sets "white-space: nowrap";
  3. It delegates text resizing operation to jquery-textfill plugin;
  4. If jquery-textfill plugin reports that text after resizing to minimal size (specified by options object) still doesn't fit container, than this widget sets "white-space: normal" css style, allowing wrapping lines on white spaces, and sets "font-size" css style to minimal value.

This widget created because jquery-textfill plugin does resize after text is wrapped by presence of "white-space: normal" css property (default value).

Parameters:
Name Type Description
options object the same options object as in jquery-textfill plugin
